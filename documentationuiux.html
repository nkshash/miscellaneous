Database Documentation - Detailed

This document provides an in-depth description of each database collection used in the application.

1. User Collection

Purpose: The User collection stores account information for each registered user. It contains personal details and preferences.

Fields:

  • _id: Unique identifier for the user document.
  • first_name: User's first name.
  • last_name: User's last name.
  • username: Chosen username for the account.
  • email: User's email address, used as a unique identifier.
  • gender: User's gender.
  • password: Hashed password for authentication.
  • active: Indicates whether the account is active.
  • illness, allergies, medication: Additional health-related details.
  • profile_picture: URL of the user's profile picture.
  {
      "_id": "66e3339855827b77961981c0",
      "first_name": "Shashank",
      "last_name": "Mishra",
      "username": "shawshank",
      "email": "abc@gmail.com",
      "gender": "Male",
      "password": "pbkdf2:sha256:600000$...",
      "active": true,
      "illness": "NA",
      "allergies": "NA",
      "medication": "NA",
      "profile_picture": "http://res.cloudinary.com/.../nav5pwd5i..."
  }

2. Feedback Collection

Purpose: Captures feedback from users about their experience with the application.

Fields:

  • _id: Unique identifier for each feedback entry.
  • email: Email of the user providing feedback.
  • csat: Customer Satisfaction rating (1â€“5).
  • feedback_text: Additional comments or feedback.
  • timestamp: Date and time when the feedback was submitted.
  {
      "_id": "66e5d42b9ca2fbb0d0988b3c",
      "email": "abc@gmail.com",
      "csat": "5",
      "feedback_text": "Noise",
      "timestamp": "2024-09-14T18:21:31.968+00:00"
  }

3. Comment Collection

Purpose: Stores user-submitted comments, often used for contacting support or leaving suggestions.

Fields:

  • _id: Unique identifier for each comment.
  • name: Name of the person leaving the comment.
  • email: Email of the commenter.
  • subject: Subject of the comment.
  • message: Content of the comment.
  {
      "_id": "66e316b1862df94a37bbcf3b",
      "name": "Shashank",
      "email": "abc@gmail.com",
      "subject": "hello",
      "message": "hello rhea"
  }

4. Prediction Collection

Purpose: Stores prediction data for images uploaded by users.

Fields:

  • _id: Unique identifier for each prediction entry.
  • email: Email of the user who uploaded the image.
  • image_url: Cloudinary URL of the uploaded image.
  • timestamp: Date and time of the prediction.
  • predictions: Object containing prediction-related details, such as session ID.
  {
      "_id": "66e53d573f8a58f1fd6e88ff",
      "email": "abc@gmail.com",
      "image_url": "http://res.cloudinary.com/.../n53fajlpi...",
      "timestamp": "2024-09-14T07:37:59.536573",
      "predictions": {
          "session_id": "20240914123256329775604"
      }
  }

5. User Chat History Collection

Purpose: Records chat sessions between users and the chatbot.

Fields:

  • _id: Unique identifier for each chat history entry.
  • email: Email of the user involved in the chat.
  • session_id: Unique identifier linking this chat history to predictions.
  • messages: Array of message objects exchanged during the chat.
  • timestamp: Date and time the chat session was recorded.
  {
      "_id": "66e53d883f8a58f1fd6e8900",
      "email": "abc@gmail.com",
      "session_id": "20240914123256329775604",
      "messages": [
          { "sender": "user", "content": "Hello!", "timestamp": "2024-09-14T07:38:10.200Z" },
          { "sender": "bot", "content": "How can I help you?", "timestamp": "2024-09-14T07:38:12.500Z" }
      ],
      "timestamp": "2024-09-14T07:38:48.200Z"
  }

API Documentation

This documentation provides an overview of the APIs used in the application for various services such as database connections, email configurations, cloud services, external APIs, and chatbot integration.

1. Database Configuration

Environment Variable:

  • URI: mongodb+srv://{uniqueID}/

Purpose: Used to connect the application to MongoDB Atlas for data storage and retrieval.

Example: Connecting to MongoDB
from pymongo import MongoClient
client = MongoClient(URI)
db = client.get_database("database-name")

2. Email Configuration

Environment Variables:

  • MAIL_SERVER: smtp.gmail.com
  • MAIL_PORT: Unique Code
  • MAIL_USE_TLS: True
  • MAIL_USERNAME: abc@gmail.com
  • MAIL_PASSWORD: Unique Code
  • MAIL_DEFAULT_SENDER: abc@gmail.com

Purpose: Used for sending emails, such as notifications and password resets.

Example: Sending an Email
from flask_mail import Mail, Message
mail = Mail(app)
msg = Message(subject="Hello", sender=MAIL_DEFAULT_SENDER, recipients=["recipient@example.com"])
msg.body = "This is a test email."
mail.send(msg)

3. Firebase Configuration

  • CRED: ./static/UniqueID.json

Purpose: Used for Firebase integration and Google and Microsoft Login.

Example: Firebase Integration
import firebase_admin
from firebase_admin import credentials
cred = credentials.Certificate(CRED)
firebase_admin.initialize_app(cred)

4. Cloudinary Configuration

Environment Variables:

  • CLOUD_NAME: UniqueID
  • API_KEY: UniqueID
  • API_SECRET: UniqueID

Purpose: Used for image uploads and storage in Cloudinary.

Example: Uploading an Image
import cloudinary.uploader
response = cloudinary.uploader.upload("local-image-path.jpg", cloud_name=CLOUD_NAME)
image_url = response.get("secure_url")

5. Modal API

  • cMODAL_API: https://shashankvns-test-deploy.hf.space/

Purpose: External API used for processing or image classification and predictions (e.g., ML model predictions).

Example: Calling Modal API
import requests
response = requests.post(cMODAL_API, json={"input": "data"})
result = response.json()

6. Watsonx Assistant Chatbot Integration

Configuration:

  • integrationID: UniqueID
  • region: us-south
  • serviceInstanceID: UniqueID

Purpose: Embeds a Watson Assistant chatbot on the webpage for user interaction.

Example: Embedding Watson Assistant Chat
const customElement = document.querySelector('#WebChatContainer');
window.watsonAssistantChatOptions = {
    integrationID: "2471a3ab-6e0b-4be6-a172-3304f884337b",
    region: "us-south",
    serviceInstanceID: "0c9fd8e9-11cd-4ce3-9bd8-30f3209b5ff9",
    element: customElement
};
setTimeout(() => {
    const t = document.createElement('script');
    t.src = "https://web-chat.global.assistant.watson.appdomain.cloud/versions/latest/WatsonAssistantChatEntry.js";
    document.head.appendChild(t);
});

This integration enables a chatbot interface in the webpage at the element #WebChatContainer.

UI/UX Documentation

Documentation and examples for the application's key screens and features. Screenshots and descriptions highlight the responsive behavior and interactive elements.

Landing Page

The landing page welcomes users with a clean and engaging design. It provides navigation options for login, registration, and exploring the app's features.

The profile page allows users to update personal information, including allergies and preferences. It also provides a centralized view of their account details.

The About Us page shares information about the app, its purpose, and the development team. The design is clean and focuses on building user trust.
